US-AUTH-PWD-02 — Réinitialisation mot de passe : checklist interactive

Périmètre : page /[locale]/reset-password — formulaire après lien email valide.

Mêmes règles que l'inscription (8 car., minuscule, majuscule, chiffre, caractère spécial).

Changements vs existant : remplacer Input par PasswordInput sur le 1er champ ; checklist sous « Nouveau mot de passe » uniquement ; confirmation séparée.

Submit bloqué tant que : (1) les 5 critères ne sont pas verts ET (2) les deux mots de passe ne correspondent pas.

État 1 — Initial (lien valide, champs vides)

Nouveau mot de passe

Définissez votre nouveau mot de passe

  • Au moins 8 caractères
  • Une lettre minuscule
  • Une lettre majuscule
  • Un chiffre
  • Un caractère spécial
Transition : saisie nouveau mot de passe → État 2 ; saisie confirmation → État 3 ou 4.

État 2 — Nouveau mot de passe valide, confirmation vide

  • Au moins 8 caractères
  • Une lettre minuscule
  • Une lettre majuscule
  • Un chiffre
  • Un caractère spécial
Transition : confirmation identique → État 3 ; confirmation différente → État 4.

État 3 — Nominal (critères OK + confirmation identique)

  • Au moins 8 caractères
  • Une lettre minuscule
  • Une lettre majuscule
  • Un chiffre
  • Un caractère spécial

✓ Les mots de passe correspondent

Transition : clic bouton → action serveur ; succès → message vert + redirection dashboard (existant).

État 4 — Confirmation différente (critères OK)

  • Au moins 8 caractères
  • Une lettre minuscule
  • Une lettre majuscule
  • Un chiffre
  • Un caractère spécial

Les deux mots de passe doivent être identiques

Note : le message de non-correspondance s'affiche après saisie dans le 2e champ (pas au 1er caractère). Le bouton reste désactivé — pas de submit client.

État 5 — Erreur serveur Supabase (fallback)

  • Au moins 8 caractères
  • Une lettre minuscule
  • Une lettre majuscule
  • Un chiffre
  • Un caractère spécial

Le mot de passe ne respecte pas tous les critères.

États hors formulaire (inchangés) : lien invalide → message + redirection login ; vérification lien → spinner texte « Vérification du lien… ».

Composants : PasswordInput × 2, PasswordRequirements sous le 1er champ, Button avec disabled + état loading existant.
