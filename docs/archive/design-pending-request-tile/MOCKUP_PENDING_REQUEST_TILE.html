Modales de confirmation

Au clic sur Refuser ou Accepter, une modale de confirmation s'ouvre avant d'envoyer l'action. Composant Modal (taille sm), structure header / corps / footer du design system.

Refuser la demande ?

La demande de cet athlète sera refusée. Il pourra envoyer une nouvelle demande plus tard s'il le souhaite.

Accepter la demande ?

L'athlète sera ajouté à vos athlètes et une souscription sera créée. Vous pourrez lui assigner des entraînements et objectifs.

Aligné sur components/Modal.tsx et docs/DESIGN_SYSTEM.md (Modal, taille sm, header px-6 py-4 border-b border-stone-100 bg-stone-50/50, titre text-lg font-bold, corps px-6 py-4, footer px-6 py-4 border-t border-stone-100 bg-stone-50/50, boutons flex gap-3 w-full avec Button variant="muted" + variant="danger" ou variant="primary" avec className="flex-1").

Comportement

  • Espace au-dessus du message : mt-2 pt-2 (réduit) entre la ligne infos et le bloc description.
  • Bouton Discuter : libellé « Discuter » uniquement ; style secondaire (Button variant="secondary" : bordure stone, hover bg-stone-100).
  • Bloc description : s'étale sur toute la largeur de la tuile, avec border-t border-stone-100.
  • sm et plus : ligne 1 = avatar, nom + sports + offre, boutons à droite (Discuter, Refuser, Accepter) ; ligne 2 = message en pleine largeur.
  • Petit écran : boutons tout en bas de la tuile : Discuter en pleine largeur, puis Refuser et Accepter sur une même ligne en 50/50.
  • Modales de confirmation : au clic sur Refuser ou Accepter, ouvrir une modale de confirmation avant d'appeler respondToCoachRequest. Titre + message explicatif + boutons Annuler (ferme la modale) et Refuser / Accepter (confirme l'action).

Tuiles sport = composant Badge. Avatar = AvatarImage. Boutons = Button (secondary pour Discuter, danger pour Refuser, primary pour Accepter).

Design system : Badge (sport), AvatarImage, Button (outline, muted, primary), TileCard / ActivityTile (bordure). Référence : docs/DESIGN_SYSTEM.md, lib/sportStyles.ts, components/Badge.tsx.
