La moitié haute affiche la synthèse Objectif / Réalisé (US1) — la carte Réalisé reflète les valeurs actual_* en live (preview pendant la saisie ou affichage sauvegardé). La moitié basse conserve la section retour existante : statut, saisie actual_*, feedback (édition via WorkoutFeedbackSection), commentaire, bouton Enregistrer.

Transitions fonctionnelles couvertes par US4

  • État A → État B : athlète clique Réalisé → la section saisie actual_* + feedback apparaît. Dès qu'il saisit ≥ 1 valeur → la carte Réalisé hero apparaît à côté de la carte Objectif (md+) ou en dessous (mobile) avec live preview des écarts.
  • État B (live preview) → État B sauvegardé : athlète clique Enregistrer → confirmation ✓ Enregistré → modale se ferme. À la prochaine ouverture, même rendu mais valeurs persistées.
  • État B → État C : athlète change pour Non réalisé → la carte Réalisé disparaît, les inputs actual_* + feedback se masquent, l'athlète peut sauvegarder avec un commentaire.
  • État B → État A : athlète repasse à Planifié → carte Réalisé disparaît, inputs actual_* + feedback masqués.
  • Si actual_* tous vides ET statut Réalisé : carte Réalisé masquée (cohérent US1 État E). Bouton Enregistrer reste actif (statut + commentaire + feedback peuvent être sauvegardés).
